# System Overview
Ontology 기반 청원 + 후원 플랫폼 시스템 개요

## 1. 문서 목적

이 문서는 플랫폼의 전체 시스템 구조를 설명한다.  
목표는 다음과 같다.

- 프로젝트의 상위 아키텍처 정리
- 프론트엔드 / 백엔드 / DB / 온톨로지 모듈의 역할 정의
- 데이터 흐름과 모듈 경계 명확화
- MVP 기준 설계 방향 공유

---

## 2. 시스템 한 줄 설명

이 시스템은 사용자가 청원을 등록하면,  
시스템이 이를 자동분류하고 운영자가 검토한 후 공개하며,  
다른 사용자는 서명과 후원에 참여할 수 있고,  
운영자는 이를 대시보드에서 분석할 수 있는 시민참여 플랫폼이다.

---

## 3. 핵심 아키텍처 원칙

### 3.1 모듈 분리
- 프론트엔드, 백엔드, 공통 타입, 온톨로지 엔진을 분리한다
- UI와 도메인 로직을 혼합하지 않는다
- 자동분류 로직은 독립 모듈로 유지한다

### 3.2 확장 가능성
- MVP는 단순하게 구현하되, 추후 검색엔진, 실제 결제, 고급 AI 분류로 확장 가능해야 한다
- 후원 결제는 provider abstraction 구조를 사용한다

### 3.3 일관성
- 타입 정의 일관성 유지
- API 계약과 DB 스키마의 동기화
- 카테고리와 온톨로지 사전은 중앙 관리

### 3.4 운영 중심 설계
- 자동화보다 운영 개입 가능 구조를 우선한다
- review queue는 필수다
- 통계는 개인 식별이 아닌 집계 중심이다
